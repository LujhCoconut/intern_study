## tl::expected 深度解析

### 1.1 类型本质与内存布局

`tl::expected<T, E>` 是一个**联合体（union）包装器**，在任何时候只可能包含 `T`（值）或 `E`（错误）其中之一。

```cpp
namespace tl {

template<typename T, typename E>
class expected {
private:
    // 内部存储：联合体 + 标志位
    union {
        T value_;           // 成功值
        E error_;           // 错误值
    };
    
    bool has_value_;        // 标志位：是否有值
    
public:
    // 构造成功值
    expected(T value) : value_(std::move(value)), has_value_(true) {}
    
    // 构造错误值
    expected(unexpect_t, E error) : error_(std::move(error)), has_value_(false) {}
    
    ~expected() {
        if (has_value_) {
            value_.~T();    // 析构值
        } else {
            error_.~E();    // 析构错误
        }
    }
};

// 辅助类型：用于构造错误值
struct unexpect_t {};
inline constexpr unexpect_t unexpect{};

} // namespace tl
```

**内存布局示意图：**

```
┌─────────────────────────────────────┐
│          expected<T, E>             │
├─────────────────────────────────────┤
│  ┌─────────────────────────────┐    │
│  │         union               │    │
│  │  ┌─────────────────────┐    │    │
│  │  │       value_        │    │    │  ← T 类型（当 has_value_ = true）
│  │  │    [sizeof(T)]      │    │    │
│  │  └─────────────────────┘    │    │
│  │  ┌─────────────────────┐    │    │
│  │  │       error_        │    │    │  ← E 类型（当 has_value_ = false）
│  │  │    [sizeof(E)]      │    │    │
│  │  └─────────────────────┘    │    │
│  └─────────────────────────────┘    │
├─────────────────────────────────────┤
│      has_value_ (bool, 1 byte)      │  ← 标志位
├─────────────────────────────────────┤
│         [可能的填充字节]              │  ← 内存对齐
└─────────────────────────────────────┘

总大小 ≈ max(sizeof(T), sizeof(E)) + 对齐填充
```

### 1.2 标准库集成与演进

`tl::expected` 源自 C++23 标准库 `<expected>`，但为了兼容 C++17/C++20，Mooncake 等项目使用第三方实现。

```cpp
// C++23 标准库版本（C++23 起可用）
#include <expected>
std::expected<int, Error> result;

// 第三方实现（C++17/20 可用）
#include <tl/expected.hpp>  // 或 <ylt/util/tl/expected.hpp>
tl::expected<int, Error> result;
```

**C++23 中的扩展功能：**

```cpp
// C++23 支持 std::unexpected 类型
std::expected<int, std::string> compute() {
    if (failed) {
        return std::unexpected("error message");  // 构造错误
    }
    return 42;  // 构造成功值
}

// 与异常互操作
int value = result.value();  // 无值时抛出 bad_expected_access
int value = result.value_or(0);  // 无值时返回默认值
```

### 1.3 完整 API 与使用模式

#### 1.3.1 基础构造与访问

```cpp
#include <ylt/util/tl/expected.hpp>
#include <string>

enum class ErrorCode {
    OK = 0,
    NOT_FOUND = 1,
    TIMEOUT = 2,
    NO_MEMORY = 3
};

// 构造成功值
tl::expected<int, ErrorCode> ok_result = 42;
tl::expected<int, ErrorCode> ok_result2 = tl::expected<int, ErrorCode>(42);

// 构造错误值
tl::expected<int, ErrorCode> err_result = 
    tl::unexpected<ErrorCode>(ErrorCode::NOT_FOUND);

// 检查与访问
void process_result(const tl::expected<int, ErrorCode>& result) {
    // 方式 1：显式检查
    if (result.has_value()) {
        int val = result.value();
        std::cout << "Success: " << val << std::endl;
    } else {
        ErrorCode err = result.error();
        std::cout << "Error: " << static_cast<int>(err) << std::endl;
    }
    
    // 方式 2：隐式布尔转换（C++23）
    if (result) {
        int val = *result;  // 解引用运算符重载
    }
    
    // 方式 3：带默认值的访问
    int val = result.value_or(0);  // 失败时返回 0
    
    // 方式 4：强制解引用（无值时抛出异常）
    try {
        int val = result.value();  // 失败时抛出 bad_expected_access
    } catch (const tl::bad_expected_access<ErrorCode>& e) {
        ErrorCode err = e.error();
    }
}
```

#### 1.3.2 函数式编程接口

```cpp
class DataProcessor {
public:
    // 链式操作：and_then（成功时继续）
    tl::expected<ProcessedData, ErrorCode> process() {
        return load_data()
            .and_then([](const RawData& data) {
                // 只有成功时才调用
                return validate(data);
            })
            .and_then([](const ValidData& data) {
                return transform(data);
            })
            .and_then([](const TransformedData& data) {
                return save(data);
            });
    }
    
    // 错误恢复：or_else（失败时处理）
    tl::expected<int, ErrorCode> compute_with_retry() {
        return try_compute()
            .or_else([](ErrorCode err) -> tl::expected<int, ErrorCode> {
                if (err == ErrorCode::TIMEOUT) {
                    return retry_compute();  // 重试
                }
                return tl::unexpected(err);  // 其他错误直接传递
            });
    }
    
    // 值转换：map（成功时映射）
    tl::expected<std::string, ErrorCode> get_name_upper() {
        return get_name()
            .map([](const std::string& name) {
                std::string upper = name;
                std::transform(upper.begin(), upper.end(), upper.begin(), ::toupper);
                return upper;
            });
    }
    
    // 错误转换：map_error
    tl::expected<int, std::string> compute_with_message() {
        return compute()
            .map_error([](ErrorCode code) {
                return error_code_to_string(code);
            });
    }
};
```

### 1.5 性能特性与优化

```cpp
// 性能对比：expected vs 异常 vs 指针

// 1. expected - 值语义，无运行时开销
tl::expected<int, Error> func_expected() {
    if (condition) return 42;
    return tl::unexpected(Error::Fail);
}
// 编译后：直接返回结构体，无动态分配

// 2. 异常 - 有栈展开开销
int func_exception() {
    if (condition) return 42;
    throw std::runtime_error("fail");  // 需要栈展开
}

// 3. 指针/输出参数 - 需要动态分配或预先分配
Error func_pointer(int* out) {
    if (condition) {
        *out = 42;
        return Error::OK;
    }
    return Error::Fail;
}
```

**性能数据（典型场景）：**

| 机制 | 成功路径 | 失败路径 | 内存分配 | 可预测性 |
|------|---------|---------|---------|---------|
| `expected` | 内联，无开销 | 内联，无开销 | 无 | 高 |
| 异常 | 无开销（零成本异常） | 栈展开，高开销 | 可能 | 低 |
| 指针 | 间接访问 | 间接访问 | 可能需要 | 中 |

**编译器优化：**
- 现代编译器可以对 `expected` 进行**返回值优化（RVO）**
- 小对象（如 `expected<int, int>`）可以完全内联到寄存器

## std::shared_lock 深度解析

### 2.1 互斥量类型体系

```cpp
#include <mutex>
#include <shared_mutex>

// 基础互斥量（独占）
std::mutex                    // 基本互斥
std::recursive_mutex          // 可递归重入
std::timed_mutex              // 支持超时
std::recursive_timed_mutex    // 递归 + 超时

// 共享互斥量（读写锁）
std::shared_mutex             // C++17，共享/独占
std::shared_timed_mutex       // C++14，共享 + 超时
```

**锁类型与互斥量的匹配：**

| 锁类型 | 可锁定的互斥量 | 模式 |
|--------|--------------|------|
| `std::lock_guard` | 任何基础互斥量 | 独占 |
| `std::unique_lock` | 任何互斥量 | 独占/延迟/超时 |
| `std::shared_lock` | `shared_mutex` / `shared_timed_mutex` | 共享（读） |
| `std::scoped_lock` | 多个互斥量（C++17） | 独占 |

### 2.2 std::shared_mutex 的内部实现

```cpp
// 简化的实现概念（非真实代码）
class shared_mutex {
private:
    std::mutex internal_mutex_;          // 保护内部状态
    std::condition_variable cv_;         // 条件变量
    
    // 状态计数
    size_t shared_count_ = 0;            // 当前共享锁数量
    bool exclusive_locked_ = false;      // 是否有独占锁
    size_t exclusive_waiting_ = 0;       // 等待独占锁的线程数
    
public:
    // 共享锁（读锁）
    void lock_shared() {
        std::unique_lock<std::mutex> lock(internal_mutex_);
        // 等待：不能有独占锁，且没有独占等待者（可选策略）
        cv_.wait(lock, [this] {
            return !exclusive_locked_ && exclusive_waiting_ == 0;
        });
        ++shared_count_;
    }
    
    void unlock_shared() {
        std::unique_lock<std::mutex> lock(internal_mutex_);
        --shared_count_;
        if (shared_count_ == 0) {
            cv_.notify_one();  // 唤醒等待的独占锁
        }
    }
    
    // 独占锁（写锁）
    void lock() {
        std::unique_lock<std::mutex> lock(internal_mutex_);
        ++exclusive_waiting_;
        cv_.wait(lock, [this] {
            return !exclusive_locked_ && shared_count_ == 0;
        });
        --exclusive_waiting_;
        exclusive_locked_ = true;
    }
    
    void unlock() {
        std::unique_lock<std::mutex> lock(internal_mutex_);
        exclusive_locked_ = false;
        cv_.notify_all();  // 唤醒所有等待的共享锁
    }
};
```

### 2.3 std::shared_lock 的 RAII 封装

```cpp
// 简化的 shared_lock 实现概念
template<typename Mutex>
class shared_lock {
private:
    Mutex* mtx_ = nullptr;
    bool owns_ = false;
    
public:
    // 立即加锁
    explicit shared_lock(Mutex& m) : mtx_(&m), owns_(true) {
        m.lock_shared();
    }
    
    // 延迟加锁
    explicit shared_lock(Mutex& m, std::defer_lock_t) 
        : mtx_(&m), owns_(false) {}
    
    // 尝试加锁
    explicit shared_lock(Mutex& m, std::try_to_lock_t) 
        : mtx_(&m), owns_(m.try_lock_shared()) {}
    
    // 带超时
    template<typename Rep, typename Period>
    shared_lock(Mutex& m, const std::chrono::duration<Rep, Period>& timeout)
        : mtx_(&m), owns_(m.try_lock_shared_for(timeout)) {}
    
    // 解锁
    ~shared_lock() {
        if (owns_) {
            mtx_->unlock_shared();
        }
    }
    
    // 手动加锁/解锁
    void lock() {
        if (!owns_) {
            mtx_->lock_shared();
            owns_ = true;
        }
    }
    
    void unlock() {
        if (owns_) {
            mtx_->unlock_shared();
            owns_ = false;
        }
    }
    
    bool owns_lock() const { return owns_; }
    explicit operator bool() const { return owns_; }
    
    // 禁止拷贝
    shared_lock(const shared_lock&) = delete;
    shared_lock& operator=(const shared_lock&) = delete;
    
    // 允许移动
    shared_lock(shared_lock&& other) noexcept;
    shared_lock& operator=(shared_lock&& other) noexcept;
};
```

### 2.4 在 Mooncake 中的应用场景

```cpp
// mooncake-store/include/pyclient.h

class ClientRequester {
private:
    // 保护客户端池的共享互斥量
    mutable std::shared_mutex client_pool_mutex_;
    
    std::shared_ptr<coro_io::client_pools<coro_rpc::coro_rpc_client>>
        client_pools_;
    
public:
    // 读取操作：多个线程可以同时调用（共享锁）
    auto get_client(const std::string& addr) const {
        std::shared_lock<std::shared_mutex> lock(client_pool_mutex_);
        return client_pools_->get_client(addr);  // 只读访问
    }
    
    // 写入操作：独占访问（ unique_lock ）
    void add_client_pool(const std::string& addr) {
        std::unique_lock<std::shared_mutex> lock(client_pool_mutex_);
        client_pools_->emplace(addr, ...);  // 修改操作
    }
    
    // 条件修改：先共享后升级（C++ 不支持直接升级，需要释放后重新获取）
    void get_or_create_client(const std::string& addr) {
        // 1. 先尝试共享锁读取
        {
            std::shared_lock<std::shared_mutex> lock(client_pool_mutex_);
            if (client_pools_->contains(addr)) {
                return (*client_pools_)[addr];
            }
        }  // 释放共享锁
        
        // 2. 不存在，获取独占锁写入
        std::unique_lock<std::shared_mutex> lock(client_pool_mutex_);
        // 双重检查（可能其他线程已创建）
        if (!client_pools_->contains(addr)) {
            client_pools_->emplace(addr, ...);
        }
        return (*client_pools_)[addr];
    }
};
```

### 2.5 锁升级问题与解决方案

**C++ 不支持直接的锁升级**（共享锁 → 独占锁），需要采用以下模式：

```cpp
class DataStore {
    std::shared_mutex mtx_;
    std::unordered_map<std::string, Data> data_;
    
public:
    // 模式 1：先读后写（可能有竞争条件）
    Data get_or_create_v1(const std::string& key) {
        // 读阶段
        {
            std::shared_lock lock(mtx_);
            auto it = data_.find(key);
            if (it != data_.end()) {
                return it->second;
            }
        }  // 释放读锁
        
        // 写阶段（可能有其他线程在此期间写入）
        std::unique_lock lock(mtx_);
        // 必须再次检查！
        auto it = data_.find(key);
        if (it != data_.end()) {
            return it->second;
        }
        data_[key] = create_data();
        return data_[key];
    }
    
    // 模式 2：直接使用独占锁（简单但并发低）
    Data get_or_create_v2(const std::string& key) {
        std::unique_lock lock(mtx_);
        auto it = data_.find(key);
        if (it != data_.end()) {
            return it->second;
        }
        data_[key] = create_data();
        return data_[key];
    }
    
    // 模式 3：使用 try_lock 优化（先尝试共享，失败再独占）
    Data get_or_create_v3(const std::string& key) {
        // 尝试快速路径：共享锁
        {
            std::shared_lock lock(mtx_);
            auto it = data_.find(key);
            if (it != data_.end()) {
                return it->second;
            }
        }
        
        // 慢速路径：独占锁
        std::unique_lock lock(mtx_);
        auto it = data_.find(key);
        if (it != data_.end()) {
            return it->second;
        }
        data_[key] = create_data();
        return data_[key];
    }
};
```

**使用陷阱与最佳实践：**

```cpp
// ❌ 错误：在持有共享锁时修改数据
void bad_read(Data& data) {
    std::shared_lock lock(mtx);
    data.counter++;  // 未定义行为！共享锁不保证独占访问
}

// ✅ 正确：修改使用独占锁
void good_write(Data& data) {
    std::unique_lock lock(mtx);
    data.counter++;
}

// ❌ 错误：递归获取共享锁（std::shared_mutex 不支持递归）
void bad_recursive() {
    std::shared_lock lock1(mtx);
    std::shared_lock lock2(mtx);  // 死锁！
}

// ✅ 正确：使用 std::shared_recursive_mutex（如果需要）
void good_recursive() {
    std::shared_lock lock1(recursive_mtx);
    std::shared_lock lock2(recursive_mtx);  // OK
}

// ❌ 错误：持有共享锁时等待其他资源（可能导致写者饥饿）
void bad_wait() {
    std::shared_lock lock(mtx);
    std::this_thread::sleep_for(1s);  // 长时间持有读锁
}

// ✅ 正确：减少锁持有时间
void good_quick() {
    Data local_copy;
    {
        std::shared_lock lock(mtx);
        local_copy = data;
    }  // 尽快释放锁
    process(local_copy);  // 处理在锁外进行
}

// ❌ 错误：优先级反转（写者饥饿）
// 如果有持续的读者流，写者可能永远等待
// 解决方案：使用公平锁或限制读者数量
```

### 2.7 与条件变量结合使用

```cpp
class BlockingQueue {
    std::shared_mutex mtx_;
    std::condition_variable_any cv_;  // 注意：condition_variable_any
    std::queue<Data> queue_;
    bool shutdown_ = false;
    
public:
    // 生产者：独占锁
    void push(const Data& data) {
        std::unique_lock lock(mtx_);
        queue_.push(data);
        cv_.notify_one();
    }
    
    // 消费者：可以共享锁（如果允许多消费者）
    Data pop() {
        std::unique_lock lock(mtx_);  // 需要修改队列，使用独占锁
        cv_.wait(lock, [this] {
            return !queue_.empty() || shutdown_;
        });
        
        if (shutdown_) throw ShutdownException();
        
        Data data = queue_.front();
        queue_.pop();
        return data;
    }
    
    // 查询大小：共享锁
    size_t size() const {
        std::shared_lock lock(mtx_);
        return queue_.size();
    }
    
    // 关闭队列
    void shutdown() {
        std::unique_lock lock(mtx_);
        shutdown_ = true;
        cv_.notify_all();
    }
};
```

```cpp
    auto result = process(parsed);
    display(result);
} catch (const NetworkException& e) {
    log_error("Network: %s", e.what());
} catch (const ParseException& e) {
    log_error("Parse: %s", e.what());
} catch (const ProcessException& e) {
    log_error("Process: %s", e.what());
}

// 方式 2：expected 处理
fetch_network_data()
    .and_then(parse_data)
    .and_then(process)
    .map(display)
    .or_else([](Error e) {
        log_error("Failed: %s", to_string(e));
        return tl::unexpected(e);
    });

// 方式 3：错误码返回（C 风格）
Error err;
Data data;
if ((err = fetch_network_data(&data)) != Error::OK) {
    log_error("Network: %d", err);
    return;
}
Parsed parsed;
if ((err = parse_data(data, &parsed)) != Error::OK) {
    log_error("Parse: %d", err);
    return;
}
// ...
```

### 3.2 锁选择决策树

```
是否需要并发读？
├── 否 → 使用 std::mutex + std::lock_guard/unique_lock
└── 是 → 读多写少？
    ├── 是 → 使用 std::shared_mutex + shared_lock/unique_lock
    │        注意：检查是否有锁升级需求
    ├── 否（读写平衡）→ 考虑 std::mutex（实现更简单）
    └── 不确定 → 先使用 std::mutex，必要时优化

特殊情况：
- 需要超时 → 使用 *_timed_mutex
- 需要递归 → 使用 std::recursive_mutex
- 需要多锁同时持有 → 使用 std::scoped_lock (C++17)
```

### 3.3 完整示例：线程安全的缓存实现

```cpp
#include <shared_mutex>
#include <unordered_map>
#include <optional>
#include <ylt/util/tl/expected.hpp>

enum class CacheError {
    NotFound,
    Expired,
    NoMemory
};

template<typename K, typename V>
class ConcurrentCache {
    mutable std::shared_mutex mutex_;
    std::unordered_map<K, std::pair<V, std::chrono::steady_clock::time_point>> data_;
    size_t max_size_;
    
public:
    // 获取值（共享锁）
    tl::expected<V, CacheError> get(const K& key) const {
        std::shared_lock lock(mutex_);
        
        auto it = data_.find(key);
        if (it == data_.end()) {
            return tl::unexpected(CacheError::NotFound);
        }
        
        if (it->second.second < std::chrono::steady_clock::now()) {
            return tl::unexpected(CacheError::Expired);
        }
        
        return it->second.first;
    }
    
    // 插入值（独占锁）
    tl::expected<void, CacheError> put(const K& key, const V& value, 
                                       std::chrono::seconds ttl) {
        std::unique_lock lock(mutex_);
        
        if (data_.size() >= max_size_ && !data_.contains(key)) {
            return tl::unexpected(CacheError::NoMemory);
        }
        
        auto expiry = std::chrono::steady_clock::now() + ttl;
        data_[key] = {value, expiry};
        return {};
    }
    
    // 查询大小（共享锁）
    size_t size() const {
        std::shared_lock lock(mutex_);
        return data_.size();
    }
    
    // 清理过期项（独占锁）
    void cleanup() {
        std::unique_lock lock(mutex_);
        auto now = std::chrono::steady_clock::now();
        for (auto it = data_.begin(); it != data_.end();) {
            if (it->second.second < now) {
                it = data_.erase(it);
            } else {
                ++it;
            }
        }
    }
};
```

## 总结

### tl::expected 要点

1. **设计目的**：替代异常和错误码，提供类型安全的错误处理
2. **内存效率**：union 存储，无额外内存开销
3. **函数式接口**：`and_then`, `or_else`, `map` 支持链式操作
4. **适用场景**：预期内的错误、性能敏感代码、跨边界接口

### std::shared_lock 要点

1. **设计目的**：实现读写锁，优化读多写少场景
2. **性能优势**：多线程并发读，无竞争开销
3. **使用限制**：C++ 不支持锁升级，注意写者饥饿问题
4. **适用场景**：配置表、缓存、元数据管理等读多写少场景

### 两者结合

在 Mooncake 等高性能分布式系统中，这两个工具经常一起使用：
- `shared_lock` 保护共享状态的并发访问
- `expected` 处理操作结果的类型安全传递

这种组合提供了**高性能**和**类型安全**的平衡。
