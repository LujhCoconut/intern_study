```python

    ↓
operation = StorageOperation(host_indices, token_ids, hash_value, ...)
self.backup_queue.put(operation)        # 放入备份队列
返回 operation.id

═══════════════════════════════════════════════════════════════════════

【后台线程】backup_thread_func()

while not stop:

    _page_backup(operation)

            ↓
```

### 第 3 层: MooncakeStore (Python - SGLang 侧)

**文件**: `python/sglang/srt/mem_cache/storage/mooncake_store/mooncake_store.py`

```python
batch_set_v1(keys, host_indices, extra_info)
    ↓
# 1. 添加额外标签前缀 (如果有)
if self.extra_backend_tag:
    keys = [f"{prefix}_{key}" for key in keys]
    ↓
# 2. 预处理: 构建 key 列表、指针列表、大小列表
key_strs, buffer_ptrs, buffer_sizes = _batch_preprocess(keys, host_indices)
    ↓
# MHA 模型: 每个 key 生成 k 和 v 两个 key
#   key_strs = [f"{key}_{tp_rank}_{pp_rank}_k",
#              f"{key}_{tp_rank}_{pp_rank}_v", ...]
# MLA 模型: 每个 key 只生成一个 key
#   key_strs = [f"{key}_{pp_rank}_k", ...]
    ↓
# 3. 检查 key 是否已存在 (去重)
exist_result = self._batch_exist(key_strs)
    ↓
# 4. 只写入不存在的 key
put_results = self._put_batch_zero_copy_impl(
    set_keys, set_buffer_ptrs, set_buffer_sizes
)
    ↓
调用: self.store.batch_put_from(keys, buffer_ptrs, buffer_sizes)
     (MooncakeDistributedStore - Python binding)
```

### 第 4 层: MooncakeDistributedStore (Python Binding - C++)

**文件**: `mooncake-integration/store/store_py.cpp`

```cpp
PYBIND11_MODULE(store, m) {
    .def("batch_put_from",
         [](MooncakeStorePyWrapper &self,
            const std::vector<std::string> &keys,
            const std::vector<uintptr_t> &buffer_ptrs,
            const std::vector<size_t> &sizes) {
            ↓
            std::vector<void*> buffers;  // 转换指针类型
            for (uintptr_t ptr : buffer_ptrs)
                buffers.push_back(reinterpret_cast<void*>(ptr));
            ↓
            py::gil_scoped_release release_gil;  // 释放 GIL
            return store_->batch_put_from(keys, buffers, sizes);
        })
}
    ↓
调用: PyClient::batch_put_from()  (实际是 RealClient)
```

### 第 5 层: RealClient (C++)

**文件**: `mooncake-store/src/real_client.cpp`

```cpp
RealClient::batch_put_from(keys, buffers, sizes, config)
    ↓
# 1. 检查客户端是否已初始化
if (!client_) return ErrorCode::CLIENT_NOT_INITIALIZED;
    ↓
# 2. 调用 Client (client_service.h) 的 batch put
auto results = client_->BatchPut(keys, slices_list, config)
    ↓
Client::BatchPut() -> 创建 PutOperation
    ↓
StartBatchPut(ops, config)
    ↓
SubmitTransfers(ops)   # 提交传输任务
    ↓
对于每个操作:
    - 如果是本地内存副本: TransferWrite() -> RDMA/TCP 传输
    - 如果是磁盘副本: PutToLocalFile() -> 写入本地文件
    - 如果是远端副本: RPC 调用 PutStart/PutEnd

WaitForTransfers(ops)  # 等待传输完成
    ↓
FinalizeBatchPut(ops)  # 确认写入，更新元数据
```

```
标准批量写入流程（4 阶段）

  阶段 1：StartBatchPut（申请副本位置）

  StartBatchPut(ops, client_cfg);

  做什么：

  • 向 Master 发送 BatchStartPut RPC
  • Master 根据 config.replica_num 和 config.preferred_segments 分配存储节点
  • 返回的副本位置填入 ops[i].replicas

  示例：

  请求：写 key1, key2，replica_num=2
  返回：
    key1 → [nodeA, nodeB]
    key2 → [nodeC, nodeD]

 ──────────────────────────────────────────────────────────────────────────────────────
  阶段 2：SubmitTransfers（提交传输）

  auto t0 = std::chrono::steady_clock::now();  // 开始计时
  SubmitTransfers(ops);

  做什么：

  • 遍历所有 ops
  • 对每个 op，调用 TransferEngine->submitTransfer()
  • 启动 RDMA/TCP/CXL 数据传输

  并发性：

  • 多个 key 的数据并行传输
  • 充分利用网络带宽

  ──────────────────────────────────────────────────────────────────────────────────────
  阶段 3：WaitTransfers（等待完成）

  WaitForTransfers(ops);
  auto us = std::chrono::duration_cast<std::chrono::microseconds>(...);
  if (metrics_) {
      metrics_->transfer_metric.batch_put_latency_us.observe(us);
  }

  做什么：

  • 轮询或事件等待所有传输完成
  • 记录传输延迟（用于监控）

  ──────────────────────────────────────────────────────────────────────────────────────
  阶段 4：FinalizeBatchPut（完成确认）

  FinalizeBatchPut(ops);
  return CollectResults(ops);

  做什么：

  • 向 Master 发送 BatchEndPut RPC
  • 告知 Master："数据已写入这些节点"
  • Master 更新元数据，标记对象为可用
  • 返回每个 key 的结果（成功/失败）

```

## 二、读取数据流程 (Prefetch/Get)

### 第 1 层: SGLang HiRadixCache (Python)

**文件**: `python/sglang/srt/mem_cache/hiradix_cache.py`

```python

    ↓

host_indices = self.cache_controller.mem_pool_host.alloc(prefetch_length)
    ↓
# 2. 发起预取操作
operation = self.cache_controller.prefetch(
    req_id, host_indices, new_input_tokens, last_hash, prefix_keys
)
    ↓
self.ongoing_prefetch[req_id] = (last_host_node, token_ids, host_indices,
                                  operation)
```

### 第 2 层: HiCacheController (Python)

**文件**: `python/sglang/srt/managers/cache_controller.py`

```python
prefetch(req_id, host_indices, token_ids, last_hash, prefix_keys)
    ↓
operation = PrefetchOperation(req_id, host_indices, token_ids, ...)
self.prefetch_queue.put(operation)
返回 operation

═══════════════════════════════════════════════════════════════════════

【后台线程 1】prefetch_thread_func()
    ↓
while not stop:
    operation = prefetch_queue.get()
    ↓
    # 1. 查询存储中有哪些数据命中
    hash_value, storage_hit_count = _storage_hit_query(operation)
        ↓
    batch_exists(batch_hashes)  # 批量检查存在性
    ↓
    if storage_hit_count < threshold:
        prefetch_revoke_queue.put(req_id)  # 撤销预取
    else:
        operation.hash_value = hash_value
        prefetch_buffer.put(operation)  # 交给 IO 线程

【后台线程 2】prefetch_io_aux_func()
    ↓
_page_transfer(operation)
    ↓
for batch in hash_value:
    page_get_func(operation, batch_hashes, batch_host_indices, extra_info)
        ↓
    _page_get_zero_copy(operation, hash_values, host_indices, extra_info)
        ↓
    results = storage_backend.batch_get_v1(hash_values, host_indices, ...)
        ↓
    operation.increment(success_count * page_size)  # 更新完成进度
```

## 四、调用链可视化总结

### 写入流程

```
HiRadixCache.write_backup_storage()
    ↓
HiCacheController.write_storage() → backup_queue
    ↓ (后台线程)
HiCacheController._page_backup()
    ↓
MooncakeStore.batch_set_v1()
    ↓ (MHA: 每个 key 拆分为 k,v; MLA: 每个 key 一个)
MooncakeDistributedStore.batch_put_from() [pybind11]
    ↓ (释放 GIL)
RealClient::batch_put_from() / Client::BatchPut()
    ↓
TransferWrite() [RDMA/TCP] 或 StorageBackend::StoreObject() [Disk]
    ↓
MasterService::PutEnd() [RPC 确认元数据]
```

### 读取流程

```
HiRadixCache.prefetch_from_storage()
    ↓
HiCacheController.prefetch() → prefetch_queue
    ↓ (后台线程)
HiCacheController._storage_hit_query() → batch_exists()
    ↓
HiCacheController._page_transfer()
    ↓
MooncakeStore.batch_get_v1()
    ↓
MooncakeDistributedStore.batch_get_into() [pybind11]
    ↓ (释放 GIL)
RealClient::batch_get_into() / Client::BatchGet()
    ↓
MasterService::BatchQuery() [RPC 查询元数据]
    ↓
TransferRead() [RDMA/TCP] 或 StorageBackend::LoadObject() [Disk]
    ↓
数据直接写入 host_indices 指向的 Host Memory (零拷贝)
```

### 关键设计要点

| 特性 | 说明 |
|------|------|
| **分层架构** | Python (HiCache) → Python (MooncakeStore) → pybind11 (C++) → C++ (Client) → Network (RDMA/TCP) |
| **零拷贝** | 通过 `batch_get_into`/`batch_put_from` 直接操作已注册的内存指针 |
| **异步处理** | 使用队列 + 后台线程处理存储 I/O，不阻塞主推理流程 |
| **MHA/MLA 支持** | MHA 模型每个 token 拆分为 k 和 v 两个 key，MLA 只用一个 key |
| **GIL 管理** | 在 C++ 层显式释放 Python GIL，避免阻塞其他 Python 线程 |
